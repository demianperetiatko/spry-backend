# Ідеї для інсайтів — Spry

---

## 👁️ Два режими перегляду — важливо розрізняти

### Режим 1: Ти дивишся на СЕБЕ
> Ти відкрив свою особисту аналітику. Інсайти мають допомогти **тобі самому** змінити свої звички.

Питання які ти задаєш собі:
- Чи не занадто багато часу я витрачаю на зустрічі?
- Чи є у мене час для глибокої роботи?
- Чи я добре організовую свої зустрічі?
- Як я можу звільнити більше часу?

→ Тон інсайтів: **від першої особи**, рекомендація звернена до тебе, ти можеш щось зробити прямо зараз.
Приклад: *"53% твоїх зустрічей без agenda — додай одну тему до кожної зустрічі яку ти організуєш"*

---

### Режим 2: Ти дивишся на ІНШУ ЛЮДИНУ (як менеджер)
> Ти відкрив аналітику члена своєї команди. Інсайти мають допомогти **тобі як менеджеру** прийняти рішення щодо цієї людини.

Питання які ти задаєш собі:
- Чи не перевантажена ця людина зустрічами?
- Чи є у неї час для реальної роботи?
- Чи не потрібно мені захистити її від зайвих зустрічей?
- Чи ця людина ризикує вигоріти?

→ Тон інсайтів: **від третьої особи** (ім'я людини), рекомендація звернена до тебе як менеджера.
Приклад: *"У Олени 68% часу йде на зустрічі — перевір які з них можна прибрати з її календаря"*

## 📋 Personal — інсайти для себе (режим 1)

| Інсайт | Формула | Користь |
|---|---|---|
| % часу на зустрічах | `сума тривалостей / робочий час × 100` | Розумієш чи є баланс |
| Deep work час | `робочий час - зустрічі - буфери - переходи` | Бачиш скільки часу реально для роботи |
| % зустрічей без agenda | `events де description пустий / всі × 100` | Мотивує готуватись до зустрічей |
| Великі зустрічі | `events де attendees > 5 / всі × 100` | Підказує де можна скоротити учасників |
| 1-на-1 зустрічі | `events де attendees = 2` | Баланс між груповими і особистими |
| % відмов | `response_status = declined / всі × 100` | Чи вмієш ти говорити "ні" |
| % зустрічей які я організую | `organizer_email = мій email / всі × 100` | Чи не організовуєш занадто багато |
| Вартість моїх зустрічей | `hourly_cost × сума тривалостей` | Усвідомлення реальної ціни часу |
| Буфери між зустрічами | розраховується з часових проміжків | Чи є час на перемикання |

## 👤 Personal — інсайти про людину (режим 2, менеджер дивиться)

| Інсайт | Формула | Користь менеджеру |
|---|---|---|
| % часу на зустрічах vs команда | порівняти з середнім по команді | Бачиш хто перевантажений |
| Deep work нижче норми | `deep_work < 20%` | Сигнал ризику вигорання |
| Тренд зростання зустрічей | порівняння з попереднім периодом | Раннє попередження |
| Зустрічі пізно ввечері / рано | `start_datetime поза 9-18` | Чи є work-life balance |
| Скільки зустрічей організує | `organizer_email = email людини / всі × 100` | Оцінка навантаження на організацію |
| Collaboration map | з ким ця людина найбільше зустрічається | Розумієш зв'язки і залежності |

## 👥 Teams

| Інсайт | Формула | Користь |
|---|---|---|
| Хто найбільше в зустрічах | порівняти `meetings_time %` між членами команди | Бачиш хто перевантажений |
| Найдорожча повторювана зустріч | `recurring × attendees × hourly_cost × тривалість` | Де найбільше витрачається грошей |
| Cross-team vs inside-team | attendees з інших команд vs з тієї ж | Чи команда ізольована або навпаки розпорошена |
| Хто найбільше організовує | `organizer_email` серед членів команди | Де концентрується відповідальність |
| Середній розмір зустрічей в команді | `AVG(attendees)` по всіх зустрічах команди | Чи є культура великих зборів |
| % зустрічей без agenda в команді | по всіх членах команди | Культура підготовки до зустрічей |

## 🏢 Organization

| Інсайт | Формула | Користь |
|---|---|---|
| Топ-2 найдорожчих зустрічей | recurring зустрічі відсортовані по вартості | Де найбільший потенціал для економії |
| % зустрічей без agenda по всій орг | глобально по всіх members | Загальна культура організації |
| Тренд (зростання/падіння) | порівняння поточного і попереднього периоду | Чи стає краще або гірше |
| Найбільший організатор | хто організовує найбільше зустрічей в org | Де концентрується координаційне навантаження |
| Загальна вартість зустрічей | сума по всій організації | Реальна ціна координації |
| Розподіл: inside / cross-team / external | % по кожному типу | Чи є здоровий баланс |
